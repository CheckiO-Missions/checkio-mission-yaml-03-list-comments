У 3-му завданнях з парсингу YAML ми будемо розглядати {% if is_js %}масиви{% else %}списки{% endif %}.

(Пам'ятай, що в цій місії ти маєш враховувати умови попередніх місій. Це означає, що твоє рішення має працювати з словниками з минулої місії і з {% if is_js %}масивами{% else %}списками{% endif %} з цієї).

Простий приклад:

- write some
- "Alex Chii"
- 89
            
[
"write some", 
"Alex Chii", 
89
]

Як ти бачиш у наведеному вище прикладі, формат для значень такий же, як ми використовували в попередньому завданні для формату значень у словнику.

Настав час додати коментарі до формату.

Все, що вказано після знака #, повинно бути ігнороване парсером.

# comment
- write some # after
# one more
- "Alex Chii #sir "
- 89 #bl
            
[
"write some", 
"Alex Chii #sir ", 
89
]

Як видно, коментарі всередині рядка не вважаються коментарями, а частиною рядка, що має сенс.

Вхідні дані: Форматований рядок.

Вихідні дані: Об'єкт ({% if is_js %}масив/словник{% else %}список/словник{% endif %}).

Приклади: 

{{init_code_tmpl}}

Передумова: Використовується стандарт YAML 1.2.

В третьей задаче по парсингу YAML мы познакомимся с массивами.

(Помните, что в этой миссии надо учитывать условия предыдущих миссий. Это значит, что ваше решение должно работать со словарями из предыдущей миссии и с массивами из этой.)

Простой пример:

- write some
- "Alex Chii"
- 89

[
  "write some", 
  "Alex Chii", 
  89
]

Как видите в этом примере - формат для значений такой же как мы использовали для формата значений в словаре, в предыдущей задаче.

Пришло время дополнить формат комментариями.

Все, что указано после знака # должно игнорироваться парсером.

# comment
- write some # after
# one mor
- "Alex Chii #sir "
- 89 #bl

[
  "write some", 
  "Alex Chii #sir ", 
  89
]

Как вы могли заметить, что комментарии внутри строки не считаются комментариями, а рассматриваются как часть строки, что логично.

Входные данные: Строка в формате

Выходные данные: Объект.

- "Alex Chii"

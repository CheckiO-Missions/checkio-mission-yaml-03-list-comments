In der 3. YAML-Parsing-Aufgabe werden wir uns mit {% if is_js %}Arrays{% else %}Listen{% endif %} beschäftigen.

(Denken Sie daran, dass Sie in dieser Mission die Bedingungen der vorherigen Missionen berücksichtigen müssen. Das bedeutet, dass Ihre Lösung mit {% if is_js %}Objekten{% else %}Wörterbüchern{% endif %} aus einer früheren Mission und mit {% if is_js %}Arrays{% else %}Listen{% endif %} aus dieser Mission funktionieren sollte).

Ein einfaches Beispiel:

- write some
- "Alex Chii"
- 89
            
[
"write some", 
"Alex Chii", 
89
]

Wie Sie im obigen Beispiel sehen können, ist das Format für die Werte dasselbe, das wir in der vorherigen Aufgabe für das Format der Werte im Objekt {% if is_js %}{% else %} dictionary{% endif %} verwendet haben.

Es ist an der Zeit, das Format mit Kommentaren zu versehen.

Alles, was nach dem Zeichen # angegeben wird, sollte vom Parser ignoriert werden.

# comment
- write some # after
# one more
- "Alex Chii #sir "
- 89 #bl
            
[
"write some", 
"Alex Chii #sir ", 
89
]

Wie Sie vielleicht bemerkt haben, werden die Kommentare innerhalb der Zeichenkette nicht als Kommentare betrachtet, sondern als Teil der Zeichenkette behandelt, was durchaus sinnvoll ist.
